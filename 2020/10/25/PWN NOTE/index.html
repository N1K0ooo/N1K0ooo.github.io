PWN NOTE

本文主要介绍PWN学习过程中的各种收获和笔记

常规

  • 搜索gadgets/搜索rop链：

    • ROPgadget --binary a.out --only 'pop|ret' | grep 'eax'
    • ROPgadget --binary b0verfl0w --only 'jmp|ret'

  • ropper ropper --file /lib/x86_64-linux-gnu/libc.so.6 --search "syscall"

  • 在elf中搜索字符串：ROPgadget --binary a.out --string '/bin/sh'

  • 在ida中搜索字符串 shift + F12

  • 计算偏移量：

    cyclic 100 生成有规律的字符串

    cyclic -l abcd 根据返回值计算其所在位置

    • 编写exp时如果涉及到函数调用，要记得在调用函数和参数之间加返回值（可以是占位符），如：

      payload = 'a' * 112 + system_addr + 'aaaa' + binsh_addr

      这主要是因为我们通过栈溢出改变函数返回值的方式，其过程类似于执行了call system，但在执行call执行时，其内部会将下一个指令的地址入栈，待函数执行完毕后执行ret指令时，会从栈顶pop返回值。因此需要在修改的函数地址后面跟一个返回值，否则栈上构造的参数会被ret指令读取

    • 当遇到nx保护又想执行shellcode时，可以调用mprotect或mmap函数更改bss段的权限

    • 查看程序符号表入口（可查看自定义名称的函数） objdump -t level5

    • 查看程序got表 objdump -T level5

    • libc搜索网站和工具（搜索后12位）：LibcSearcher、DynELF、https://libc.blukat.me/

    • onegadget： 一个神奇的工具 用来找libc中能一个gadget就get到shell的偏移

      one_gadget /lib/x86_64-linux-gnu/libc.so.6

    • 没有开启PIE时，64位程序从0x400000开始

    • 翻译汇编：
      sub_esp_jmp = asm('sub esp, 0x28;jmp esp')

    • 一种调试方法：

      def DEBUG(cmd):
	raw_input("DEBUG: ")
	gdb.attach(io, cmd) 
	
DEBUG('b *0x1234 \n c') #在0x123处下断点，然后继续执行至断点处

    • byte ptr [eax]

      以eax为地址，取一个字节的数据，并扩展到32位

    • retn指令 = pop eip 将栈上的地址赋值给eip

    • 调试开启PIE的程序：

      • 下断点：

        pwndbg> b main (首先在main处下断点，让程序跑起来)

      pwndbg> b *$rebase(0xAAAA)

      • 查看内存：

        pwndbg> x/xg $rebase(0xAAAA)

  • gdb

    • 字符串 x/s 0xxxx
    • 指令 x/i 0xxxx

  • ida注释

    • 汇编指令注释：’;’
    • 伪代码注释：’/‘

  • 字符串格式化漏洞算偏移

    • fmtarg + 栈上地址

  • 在vscode中开启 pwntools gdb调试

    • 原来的代码是直接attach调用：

      p = process('./dice_game')
gdb.attach(p)

      现在修改为：

      p = process('./dice_game')
context.terminal = ['tmux','splitw','-h']
gdb.attach(proc.pidof(p)[0],gdbscript="b main")
      • 运行脚本的时候，先配置tmux：
      • tmux set mouse on （设定鼠标滚动支持）
      • 再执行 tmux

      再输入python exploit.py，**[^记得务必在tmux下执行脚本]** ，然后就可以发现使用gdb进行debug了，整个命令行界面分为一左一右两部分。

小Trick

  • 去除pwn题中alarm函数

    • sed -i s/alarm/isnan/g ./xxx

  • 泄露栈地址的方法

    • 首先通过__libc_start_main,puts等获得libc基址
    • 之后通过泄露_environ + base获得栈地址

  • 32和64位下各类型变量长度

    • 64位

    • 32位

  • system(‘/bin/sh’) == system(‘asdasdsdas;/bin/sh’)

    • ;会截断system的参数，因此可以使诸如sadjljashd;/bin/sh的字符串成功get shell

  • gdb使用特定格式查看内存

    • pwndbg> p *(const struct _IO_jump_t *) IO_2_1_stdout.vtable

  • 开启沙箱保护的特征：

    • 存在seccomp 或 prctl函数
      • 可以用seccomp-tools查看保护
      • seccomp-tools dump ./one_punch

  • 在本地部署pwn题调试

    安装 socat

    $ sudo apt-get install socat

    模板：socat tcp-listen:port,fork exec:./程序名,reuseaddr

    实例：socat tcp-listen:10001,fork exec:./pwn1,reuseaddr
